# 第五章：开发者体验

## 5.1 开发者接入流程

AISI协议致力于为开发者提供“1小时接入”的平滑体验。无论您是希望消费智能体服务，还是将自身能力封装为原子化服务提供，都能快速开始。

### 5.1.1 快速开始（5分钟）
1.  **获取API密钥**：访问 [AISI开发者门户](https://developer.aisi.run)，注册账号并创建API密钥。
2.  **安装SDK**：选择适合您开发语言的官方SDK。
    ```bash
    # Python
    pip install aisi-sdk
    # Node.js
    npm install aisi-sdk
    # Go
    go get github.com/aisi/sdk
    ```
3.  **首次调用**：使用您的API密钥初始化客户端，并尝试发现或调用一个服务。
    ```python
    from aisi import Client
    client = Client(api_key="your_key")
    services = client.discover_services(category="algorithm")
    print(f"发现 {len(services)} 个算法服务")
    ```

### 5.1.2 一站式开发者门户
我们提供统一的开发者门户 (`developer.aisi.run`)，整合以下功能：
- **服务市场**：浏览、搜索和筛选所有已注册的原子化服务。
- **交互式沙盒**：直接在浏览器中尝试调用服务，无需编写代码。
- **个人仪表板**：查看您的API使用统计、效率验证历史和项目。
- **学习路径**：根据您的目标（如“成为服务提供者”、“优化效率”）推荐个性化教程。

## 5.2 工具链支持

### 5.2.1 命令行工具 (AISI CLI)
提供功能完整的命令行工具 `aisi`，涵盖开发全流程：
```bash
# 项目初始化
aisi init my-service
# 本地效率验证
aisi verify --local
# 部署服务到测试环境
aisi deploy --staging
# 查看服务实时日志
aisi logs my-service --follow
5.2.2 IDE集成插件

VS Code扩展：提供代码补全（智能推荐相关服务）、内联文档、一键执行和效率分析。
IntelliJ / PyCharm 插件：深度集成项目导航、运行配置和调试工具。
5.2.3 调试与诊断

交互式调试器：支持在服务调用过程中设置断点、检查变量、单步执行。
性能分析器：自动分析服务执行的性能瓶颈，并给出具体的优化建议（如“减少I/O调用”、“优化算法复杂度”）。
5.3 CI/CD与自动化

5.3.1 预置流水线模板

提供开箱即用的CI/CD配置模板（GitHub Actions, GitLab CI）。

yaml
# .github/workflows/aisi.yml 示例
name: AISI Pipeline
on: [push]
jobs:
  verify:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Run Efficiency Verification
        run: aisi verify --ci
      - name: Upload Report
        uses: actions/upload-artifact@v3
        with:
          name: efficiency-report
          path: ./aisi-report.json
5.3.2 自动化部署

支持一键将服务部署到多环境：

沙盒环境：用于个人测试，资源有限。
测试环境：用于团队集成测试。
生产环境：需通过自动化测试和效率验证阈值后方可部署。
5.4 文档与支持体系

5.4.1 分层文档

L1: 快速开始：5分钟教程、交互式示例。
L2: 核心概念指南：深入讲解效率验证、原子化服务、协同协议。
L3: API参考：完整的REST API、gRPC接口和SDK文档。
L4: 最佳实践与架构：大型项目指南、性能调优、安全合规。
5.4.2 智能文档生成

每个注册的服务都会自动生成交互式API文档页面（基于OpenAPI 3.0），包含：

可执行的代码示例（多种语言）。
参数详细说明与约束。
实时测试控制台。
5.4.3 社区支持

Discord社区：实时技术交流、公告发布。
Stack Overflow：官方标签 aisi-protocol。
GitHub Discussions：功能讨论、提案与问题反馈。
